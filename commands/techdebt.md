# Поиск и логирование технического долга

Проанализируй текущую кодовую базу и залогируй найденный tech debt.

## Шаг 1: Анализ

Проверь кодовую базу на:
- Дублированный код (повторяющиеся паттерны > 10 строк)
- TODO/FIXME/HACK комментарии в коде
- Нарушения типизации (`any` в TypeScript, отсутствие типов)
- Неиспользуемые зависимости
- Незарегистрированные/неучтённые компоненты (если проект ведёт registry)
- Устаревшие паттерны (противоречащие текущим гайдлайнам)
- Файлы > 800 строк
- Функции > 50 строк

## Шаг 2: Спросить пользователя

Для каждого найденного элемента уточни:
- Приоритет: high / medium / low
- Нужно ли добавлять в backlog

## Шаг 3: Записать в backlog

Добавь подтверждённые элементы в файл бэклога проекта (обычно BACKLOG.md) в секцию "Technical Debt":

```markdown
### TD-XXX: [Краткое описание]
**Категория**: refactoring / performance / security / testing / documentation
**Приоритет**: high / medium / low
**Контекст**: [файлы / цель]
```

Инкрементируй номер TD от последнего в файле.

## Шаг 4: Итог

Покажи сводку:
- Найдено элементов: N
- Добавлено в backlog: M
- Отложено/отклонено: K
