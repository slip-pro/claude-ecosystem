# Ecosystem Convention

Конфиги Claude Code разделены на два уровня:
- **Глобальные** (`~/.claude/`) — универсальные для всех проектов (ecosystem repo)
- **Проектные** (`.claude/`) — специфичные для конкретного проекта

## При создании нового агента, скилла, хука или правила:

1. Спроси: "Это универсально (для всех проектов) или специфично для этого проекта?"
2. Универсальное -> создавай в `~/.claude/` (ecosystem repo)
3. Проектное -> создавай в `.claude/` (project)
4. При сомнении -> спроси пользователя

## При редактировании существующего:

- Файл в `~/.claude/` -> редактируй там (ecosystem auto-updated через symlink)
- Файл в `.claude/` -> редактируй там (project-specific)
- НИКОГДА не копируй глобальный файл в проект чтобы отредактировать
  -> либо правь глобально, либо создай проектный override с другим именем

## После изменений в ecosystem-файлах:

Напомни пользователю:
```
Изменения внесены в ecosystem-репо (~/.claude/).
Не забудь закоммитить: cd <ecosystem-repo> && git add . && git commit && git push
```

## Приоритеты (Claude Code merging):

```
project .claude/ > user ~/.claude/ > plugin
```

Проектный файл с тем же именем полностью перекрывает глобальный.
