# Tech Debt Tracking

## При закрытии спринта (рефлексия)

После каждого спринта, в фазе рефлексии:

1. **Выявить техдолг** — из кода, из решений, из shortcuts, из TODO/FIXME
2. **Записать в бэклог** — каждый пункт как `TD-NNN` в секции "Технический долг":
   - Проблема (что именно)
   - Нужно (что сделать)
   - Приоритет (Высокий / Средний / Низкий)
   - Источник (какой спринт/фича породила)
3. **Обновить BACKLOG.md** — если фичи частично завершены, пометить что отложено и сослаться на TD

## Формат записи (GOALS.md)

```markdown
### TD-NNN: Краткое описание
**Проблема**: Что не так
**Нужно**: Что сделать для исправления
**Приоритет**: Высокий / Средний / Низкий
**Источник**: G-XXX Sprint N (F-NNN)
```

## Типичные источники техдолга

- Stubbed/mock функциональность (email не отправляется, webhook не вызывается)
- Отсутствующие тесты (E2E, edge cases)
- Хардкод вместо конфигурации
- TODO/FIXME в коде
- Workarounds (Unicode escapes, обходные пути)
- Частично реализованные фичи (пометка `[ ]` в BACKLOG)

## Не создавать TD для

- Запланированных фич следующих спринтов (это фичи, не долг)
- Гипотетических улучшений ("можно было бы...")
- Вещей, которые работают корректно, просто не идеально
